# 音准训练器

基于浏览器的音准训练单页应用。通过 5 个递进关卡，从零开始建立基本的音准感知与控制能力。

## 训练思路

训练围绕"听→唱→看→调"的闭环展开，逐步减少视觉辅助，最终由听觉主导判断：

1. **认识工具与音域**：熟悉钢琴窗格，通过自然发声确定个人舒适音域（低音域 / 高音域）。
2. **建立音高控制**：通过滑音和定点练习，建立主动控制音高升降和稳定的能力。
3. **锚定练习**：听参考音后用嗓音复制，逐步脱离音高线辅助，建立听觉主导的音准判断。
4. **音程感知**：训练两个音之间的距离感，练习干净的音高跳跃（邻进、隔音、随机音程）。
5. **旋律演唱**：将多个音程连接为连贯旋律，以《小星星》前两句为练习曲目，从高辅助逐步过渡到无辅助。

每关末尾附有相关乐理原理的简要介绍。

## 核心特性

- **配置化训练流程**：5 个关卡均由 `src/levels/` 中的声明式配置驱动，便于扩展与迭代。
- **实时音频**：Tone.js 负责参考音合成与播放；Pitchy 负责麦克风音高检测；EMA 平滑（α=0.35）稳定音高显示。
- **可视化反馈**：Canvas 实时绘制 15 秒滑动窗口内的音高轨迹（蓝色）、响度区域（橙色）和目标参考线。
- **音域模式**：支持低音域（C3–B3）和高音域（C4–B4）切换，运行时自动转置音符与说明文本。
- **编译时版本注入**：`vite.config.js` 在构建时读取 git tag 和 commit hash，注入到页面底部。
- **分辨率守卫**：当视口宽度 < 1024px 或高度 < 768px 时显示全屏遮罩，阻止交互。

## 技术栈

- Vue 3 + Vite（Composition API）
- Pinia（状态管理）
- Tone.js（音频合成）
- Pitchy（音高检测）
- Canvas API（音高可视化）

## 使用方式

```bash

npm install

# 启动开发服务器
npm run dev

# 生产构建
npm run build
```

## 使用建议

- 首次训练前先允许浏览器麦克风权限。
- **必须佩戴耳机**，避免扬声器参考音被麦克风录入导致音高线混乱。
- 建议在安静环境中训练，优先关注稳定性而非一次命中。
